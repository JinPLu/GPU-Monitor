# gpu-broker 实施状态

更新时间：2026-07-20（Asia/Shanghai）

## 当前状态

- 本机功能版已交付：GUI、CLI 和 MCP 共用 REST 领域逻辑，可查看、认领、排队、预约和释放 GPU。
- 默认无登录且仅监听 loopback；Collector 固定只读，异常、失效 telemetry 和非托管进程均 fail closed。
- 当前开放的是协作租约；租约本身不授权远端执行，也不启动、停止或抢占工作负载。自动 allocator 与远端生命周期管理未开放。
- GUI 支持通过原始 SSH 指令把服务器加入本机资源池；粘贴内容只解析连接信息、不执行远端命令，并拒绝管道、跳板和密钥参数。服务器删除统一走 REST，不再有 HTML/CSRF 回退；桌面端会先检查本机服务是否声明 `endpoint_deletion` 能力，旧服务会禁用删除并明确提示需要更新本机服务。删除只移除本机登记、GPU 状态、采集记录和相关审计/监控历史，不会停止远端服务器或任务；仍被活跃/历史租约、未来预约、排队请求、维护记录或启用预设任务引用的服务器会被拒绝删除。
- Windows 桌面源码构建入口已加入：PowerShell + PyInstaller 生成 `dist/windows/GPU Broker/GPU Broker.exe`，运行时写入 `%LOCALAPPDATA%\GPU Broker`，启动同一 loopback REST/Web UI，不复制调度、租约或审计规则；它仍不是已签名安装包。
- 预设任务（workload profile）是可选的持久化资源合同：明确 `profile_id` 的 GUI/MCP 认领只传配置与任务，服务在立即或排队后分配时自动激活。没有 profile 时，一次性认领需要任意非空项目标识、任务、GPU 数量，以及需要的 CPU 核数、系统内存 MiB、单卡总显存/可用显存 MiB 等绝对值下限，任务名自动记录为用途；项目标识首次使用会自动登记为中性归属标签，不需要预创建、项目管理入口或服务器项目授权。申请不再要求预计占用时间，任务完成后释放租约；最大租约窗口是调度与未来预约共同遵守的硬边界。Agent 不得根据任务、目录或空闲容量自行挑选 profile，也不得推断项目、GPU 数量或 CPU/内存/显存需求。
- Broker 现在提供面向 Agent 的共享协调看板：不需要专门调度 Agent。`gpu_coordination` 一次返回每台服务器由谁租用、项目/任务、总卡/空卡/租约卡、受管运行与空租约、未归属进程、队列压力和事实性显存/利用率；`gpu_bind_observed_workload(agent_name, lease_id)` 将已启动的、当前采集到的 lease 进程登记为受管（`run_id` 可省略），不会启动或停止远端任务。采集端以小范围容忍 `ps etimes` 推算启动时刻的一秒级抖动，避免长任务因观测误差丢失归属；真实归属冲突可由租约所有者用同一动作基于当前观测自助恢复。
- macOS GUI 的资源总览已按 Apple Home 风格重构，并保留原有的侧栏、四项摘要、需要处理、操作区、服务器池和当前租约布局。主交互统一为自适应烟玫瑰，绿/橙/红只用于在线、等待和错误语义；按钮、图标、选中态、进度条与显存环共用集中 token。每台服务器使用同一张卡片模板，CPU、系统内存、GPU 利用率和显存均以对齐的进度条和绝对值呈现，每块 GPU 以显存占用环显示；服务器菜单支持复制 SSH、查看详情和经能力探测的删除。服务器池与当前租约在固定工作面内独立纵向滚动，租约展示项目、任务、操作者、GPU 数、GPU ID、到期时间和状态。窗口在宽屏显示三列服务器卡，最小 1024×640 窗口收敛为两行摘要和单列服务器卡，并隐藏次要租约字段以避免横向溢出。
- Collector 同时展示每台服务器的 CPU 核数、1 分钟负载及系统内存可用量/总量；这些观测随 REST 与 MCP 快照返回。调度会按请求中的绝对 CPU/内存/显存下限过滤候选服务器和 GPU，不满足时保持排队并 fail closed。
- 外部 Agent 的完整运行契约由 `gpu-broker` MCP instructions 和工具 schema 提供；英文全局规则是跨客户端的路由与安全适配块，明确 GPU 相关任务可主动调度 Broker、复用已批准资源合同、排队继续等待、满额安全并行、绑定观测和释放租约，同时继续禁止旁路与推断缺失输入。

## 验证快照

- 当前 macOS 桌面已完成 1440×820 与 1024×640 两种窗口尺寸的实机截图复查，并通过原生构建；删除能力探测、旧服务禁用移除提示、REST-only 删除、租约状态页、资源约束表单、绝对资源指标和新文本口径均保留。Windows launcher 的路径和默认 inventory 初始化有单元测试覆盖；Windows `.exe` 仍需在 Windows 上运行 `desktop\build-windows-app.ps1` 产出。
- 两次人工只读 shadow 和一次 5 分钟轻量化冒烟已完成；非托管进程与失效 endpoint 均正确闭锁。
- 上述结果是 2026-07-19 的历史证据，不代表当前 GPU 可用性。详细记录见 [归档](archive/IMPLEMENTATION_EVIDENCE_2026-07-19.md)。
